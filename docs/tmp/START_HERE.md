## 🚀 How to Use (Cách Sử Dụng)

### Lần Đầu Tiên (First Time)
```
1. Mở README_ROADMAP.md
2. Đọc mục "QUICK ACTIONS"
3. Xem "Current Status" để hiểu tình hình
4. Xem "Success Criteria"
```

### Lập Kế Hoạch Tuần (Plan Week)
```
1. Mở QUICK_START.md
2. Tìm phần "Tuần X: ..."
3. Đọc tasks của tuần đó
4. Ước lượng effort & resource
5. Assign cho team members
```

### Khi Cần Code Feature (Code Feature)
```
1. Mở ARCHITECTURE_GUIDE.md
2. Tìm feature cần làm
3. Copy template cấu trúc
4. Làm theo quy tắc đặt tên
5. Reference TASK_ROADMAP.md nếu cần detail
```

### Di Chuyển Files (Refactor)
```
1. Mở FILE_MIGRATION_MAPPING.md
2. Tìm file cần di chuyển
3. Note vị trí mới
4. Move files
5. Update imports
6. Run: mvn clean compile
```

### Theo Dõi Tiến Độ (Track Progress)
```
1. Mở TASK_DASHBOARD.html bằng browser
2. Scroll qua các phase
3. Click checkbox khi task done
4. Xem progress bar update
5. Share link với team
```

### Tìm Task Cụ Thể (Find Task)
```
1. Mở TASK_ROADMAP.md
2. Nhấn Ctrl+F để search
3. Gõ tên task hoặc keyword
4. Đọc chi tiết task
5. Bắt đầu code
```

### Architecture Decision
```
✅ CHOSEN: Feature-Based Architecture
   WHY: Better for team collaboration & feature management
   TEMPLATE: Provided in ARCHITECTURE_GUIDE.md
```

### Tech Stack
```
Backend: Jakarta Servlet (NOT Spring)
ORM: Hibernate 7.3
Database: MySQL 8.0+
Frontend: JSP/JSTL
Images: Cloudinary API
Build: Maven
JDK: Java 21
```
